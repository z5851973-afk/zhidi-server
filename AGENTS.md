# Codex 项目入口

开始任何任务前，必须先完整阅读根目录的 `PROJECT_STATUS.md`，再按任务需要读取相关源码。不要为了了解项目而一次性加载全部代码或全部历史计划。

工作规则：

- `PROJECT_STATUS.md` 是项目现状的快速入口；源码、测试和数据库迁移是最终事实来源。
- 开始修改前先运行 `git status --short`，保护用户已有的未提交改动。
- 只读取与当前任务直接相关的文件，优先使用 `rg` 定位符号和接口。
- 完成一项会改变项目能力、接口、目录、运行方式或完成状态的任务后，同步更新 `PROJECT_STATUS.md`。
- 不要把临时调试过程、冗长实现细节或未经验证的设想写入 `PROJECT_STATUS.md`。
- 未经用户明确要求，不修改或删除已有业务文件，不自动提交、推送或创建 PR。
