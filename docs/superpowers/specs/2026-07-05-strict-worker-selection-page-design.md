# 师傅严选页设计

## 目标

“师傅严选”页不再是一张罗列五项审核机制的宣传长图，而是一张以真实证据建立信任、并引导业主继续找师傅的产品页。

页面需要回答三个问题：平台审核了什么、审核结果为什么可信、用户接下来可以看谁。

## 设计方向

采用“证据转化型”结构。先给出严选结论，再通过一位真实在驻师傅展示审核结果，随后解释五项审核依据，最后进入本地严选师傅列表。

不采用纯品牌宣言型页面，因为只描述机制但缺少可验证对象；也不在本页展开完整施工保障闭环，避免稀释“严选审核”的主题。

## 页面结构

页面按以下顺序组织：

1. 顶部结论区：标题为“每一位师傅，都有证据可查”，简要说明身份、经验、案例和服务记录均经过平台核验。
2. 审核摘要：展示“5 项入驻审核、持续履约复核、记录可追溯”三个事实。不展示没有真实统计来源的通过率或夸张数字。
3. 真实师傅卡片：展示一位当前城市内、处于可推荐状态的真实师傅，包含头像或施工照、姓名、工种、城市、经验、服务户数、评分和已核验标签。
4. 审核档案入口：师傅卡片整体可点击，进入其个人主页；卡片内明确提供“查看师傅主页与审核档案”入口。
5. 五项核验依据：依次解释身份一致、经验与工种、真实案例、动态口碑、首单试工与抽检。每项使用短标题和一行说明，不堆叠装饰图片。
6. 平台承诺：说明资料异常、案例造假或发生严重履约问题时，平台会暂停推荐并留存可追溯记录。
7. 底部主操作：固定按钮“查看本地严选师傅”，进入按当前城市筛选的师傅列表。

## 真实数据规则

- 师傅卡片必须来自真实师傅数据，不使用仅用于讲解的虚构样例。
- 页面只展示后台存在且可追溯的字段。字段缺失时隐藏对应信息，不生成默认荣誉或虚假数值。
- “实名认证”“案例已复核”“近半年无投诉”等标签必须由明确状态计算，不能以静态文案写死。
- 评分、服务户数、从业年限和投诉状态使用与师傅详情页相同的数据源，避免两个页面口径不一致。
- 推荐师傅优先选择当前城市、资料完整、可接单且审核状态有效的对象；没有符合条件的师傅时，不显示空壳卡片。

## 空状态与异常

如果当前城市暂无可展示的严选师傅，保留审核机制和平台承诺，将示例卡替换为说明：“本地严选师傅正在审核中”，主按钮改为“查看全部可服务师傅”。

如果师傅在页面展示后被暂停推荐，进入详情前重新读取状态；不可用时给出明确提示并返回更新后的师傅列表。

## 视觉规则

- 延续业主端暖白底色与橙色品牌色，降低原稿的大面积橙色和营销海报感。
- 视觉重点依次为标题、真实师傅、审核证据和主操作。
- 真实师傅卡片使用真实头像或施工照片；五项机制使用统一线性图标，不混用证件照、施工图、评分卡等多种视觉模板。
- 内容区保持单列，控制段落长度和卡片数量，避免页面像活动落地页。
- 底部固定主操作需要避让安全区，不遮挡最后一段内容。

## 导航与行为

- 真实师傅卡片进入现有师傅详情页，并定位到可查看的认证或审核信息。
- 底部按钮进入现有师傅列表页，带入当前城市和“平台严选”筛选条件。
- 返回后保持原页面滚动位置。
- 所有可点击元素提供按压态；跳转失败时显示可理解的反馈，不静默失效。

## 验收标准

- 页面在常见手机宽度下无溢出，底部按钮不遮挡内容。
- 展示字段与师傅详情页数据一致。
- 真实师傅卡片和底部按钮均可完成正确跳转。
- 无符合条件的师傅时显示明确空状态。
- 不出现无数据依据的通过率、认证状态或案例数量。
- 页面首屏能同时看到严选结论和真实师傅卡片的主要信息。
